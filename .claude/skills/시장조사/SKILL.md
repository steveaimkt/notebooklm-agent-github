# 시장조사 OS — 4단계 파이프라인 스킬

## 스킬의 목적

리뷰 데이터에서 시작해서 다음 에이전트(상세페이지·콘텐츠·제안서)에 바로 넘길 수 있는 리서치 브리프까지 자동 생성한다.

---

## 실행 환경 전제

이 스킬을 시작하기 전에 다음이 갖춰져 있어야 한다. 갖춰지지 않은 경우 사용자에게 알리고 진행 가능 여부를 확인한다.

- 안티그래비티 작업 환경 (Part 3 세팅 완료) — 필수
- NotebookLM MCP 연동 (Part 4 클립1 완료) — 권장 (없으면 시장 트렌드 단계 스킵)
- 작업 루트에 `input/`, `output/` 디렉토리 — STEP 0-A에서 자동 탐지/생성

---

## STEP 0. 시작 시 사용자에게 받을 정보

스킬이 트리거되면 **먼저 다음 4가지를 한 번에 묻고 답을 받은 뒤** 변수에 저장한다. 절대 임의로 가정하지 말고 반드시 묻는다.

```
시장조사 스킬을 시작합니다. 진행을 위해 아래 정보를 알려주세요.

1. 분석 대상 제품명 (예: 크래프트볼트 무선 전기톱)
2. 카테고리 / 검색 키워드 (예: 무선 전기톱)
3. NotebookLM 노트북 이름 (없으면 "없음" 입력)
4. 리뷰 캡처 이미지를 함께 첨부해 주세요 (50~80개 분량 권장)
   별점 1~2점과 4~5점이 골고루 섞여 있는지 확인 필요
5. (선택) 경쟁사 검색 결과 캡처 + 시장 뉴스 본문 텍스트
```

**변수 매핑:**
- `{product_name}` — 분석 대상 제품명
- `{product_category}` — 카테고리/검색 키워드
- `{notebooklm_notebook}` — NotebookLM 노트북 이름 (또는 "없음")
- `{product_slug}` — `{product_name}`에서 영문/숫자만 추출한 slug (예: "크래프트볼트 무선 전기톱" → "craftvolt")

`{product_slug}`는 모든 파일명에 사용한다. 예: `{product_slug}_reviews.md` → `craftvolt_reviews.md`

---

## STEP 0-A. 작업 루트 결정 (자동 실행)

STEP 0 답변을 받은 직후, 산출물을 어디에 저장할지 자동으로 결정한다. 사용자에게 따로 묻지 않고 결과만 알린다 (모호할 때만 확인).

**탐지 로직:**

1. **현재 작업 디렉토리(cwd)와 1단계 하위 디렉토리**에서 `input/`과 `output/` 폴더 쌍이 함께 있는 곳을 스캔한다.
2. 결과별 처리:

| 발견 결과 | 처리 |
|---|---|
| **0개** | 사용자에게 "현재 폴더(`{cwd}`)에 `input/`, `output/`을 생성할까요? 아니면 특정 하위 폴더명을 지정하시겠어요?"라고 묻고 답을 기다린다. 답을 받으면 거기에 폴더 생성. |
| **1개** | 그 폴더를 `{work_root}`로 자동 채택하고 사용자에게 알린다 — 예: "✅ 작업 루트: `./notebooklm-agent-github/`" |
| **2개 이상** | 후보 목록을 번호로 보여주고 선택을 요청한다. |

**변수 매핑:**
- `{work_root}` — 결정된 작업 루트의 절대 경로

**이후 모든 경로는 `{work_root}` 기준으로 작성한다:**
- `{work_root}/input/{product_slug}_reviews.md`
- `{work_root}/input/market_research_data.md`
- `{work_root}/output/review_keywords_analysis.md`
- `{work_root}/output/target_personas.md`
- `{work_root}/output/market_analysis_report.md`
- `{work_root}/output/research_brief.md`

**탐지 명령 예시 (참고):**
```bash
find "$cwd" -maxdepth 2 -type d -name input | while read d; do
  parent=$(dirname "$d")
  [ -d "$parent/output" ] && echo "$parent"
done
```

## 단계별 실행 규칙

1. **각 Step 종료 후 사용자에게 결과를 요약 제시**하고, 다음 진행 여부를 묻는다.
   - "이 결과로 다음 단계 진행할까요? (진행 / 다시 / 수정 요청)"
2. 사용자가 "다시"라고 하면 같은 Step을 재실행한다.
3. 사용자가 구체적 수정 요청을 하면 그 부분만 보강해서 재출력한다.
4. NotebookLM 관련 단계는 `{notebooklm_notebook}`이 "없음"이면 스킵하고 사용자에게 알린다.
5. **사용자 검증이 반드시 필요한 게이트(★)**에서는 자동 진행하지 말고 명시적 OK를 기다린다.

# 1단계. 데이터 준비

## Step 1-1. 리뷰 이미지 → 마크다운 변환 ★

**조건:** 사용자가 리뷰 캡처 이미지를 채팅에 첨부함.

**실행:**
첨부된 이미지를 OCR로 읽어서 다음 프롬프트에 따라 작업한다.

```
첨부된 리뷰 이미지를 읽어서 input/{product_slug}_reviews.md 파일로 저장한다.
별점·작성일·리뷰 본문을 컬럼으로 갖는 마크다운 표로 만든다.
원문을 정확히 유지하고, 임의로 요약·축약하지 않는다.
```

**검증 ★:**
- 추출된 리뷰 개수 보고
- 별점 1~5 분포 보고
- 50개 미만이면 "리뷰가 부족합니다. 추가 캡처를 권장합니다"라고 알리고 사용자 결정 확인

**산출물:** `input/{product_slug}_reviews.md`

---

## Step 1-2. 시장 자료 정리 + NotebookLM 보강

**조건:** 사용자가 경쟁사 검색 결과 캡처와 시장 뉴스 본문을 첨부했을 때만 실행. 없으면 스킵.

**실행:**
```
첨부한 캡처 이미지와 뉴스 본문을 읽어서
input/market_research_data.md로 정리한다.
경쟁사 스펙은 비교표, 가격대는 분포 분석,
뉴스는 핵심 인용 + 출처 형식으로 구분해 마크다운으로 작성한다.
```

**부가 안내 (사용자 수동 작업 안내):**
> 다음 자료를 NotebookLM 노트북에 직접 추가해 주세요. URL이 아닌 **텍스트로 붙여넣기** 권장 (노이즈 제거 + 크롤링 차단 우회).
> - 정리된 경쟁사 스펙 텍스트
> - 뉴스 본문 텍스트
> - 자사 제품 설명서 PDF

**산출물:** `input/market_research_data.md`

# 2단계. 리뷰 키워드 분석

## Step 2-1. 불만 키워드 추출

**프롬프트:**
```
input/{product_slug}_reviews.md를 읽고, 고객 불만·개선 요청 키워드를 모두 뽑아라.
- 단순 감정 표현(별로, 최고)이나 배송/포장 불만은 제외
- 제품 자체에 대한 구체적 언급만 추출
- 키워드별 언급 횟수, 심각도(상중하), 대표 리뷰 원문 1개 인용
- 성능 / 품질 / 편의성 / 가격 / AS 카테고리로 분류
```

**검증 포인트:**
- 언급 횟수가 실제 카운트인가? (할루시네이션 방지)
- 인용은 원문 그대로인가?

---

## Step 2-2. 만족 키워드 + 교차 분석

**프롬프트:**
```
같은 리뷰 파일에서 만족 키워드 상위 10개를 같은 형식으로 정리하라.
이어서 불만/만족 키워드를 교차 비교해서
이 제품에서 고객이 가장 중요하게 생각하는 가치 인사이트 3가지를 도출하라.
사용 맥락의 차이(예: 무거움 vs 가벼움)가 있으면 명시적으로 짚어라.
```

**중요:** 단순 키워드 나열에서 멈추지 말고, **인사이트 단계까지 도달**해야 한다.

---

## Step 2-3. NotebookLM 시장 트렌드 + 1단계 파일 저장 ★

**조건:** `{notebooklm_notebook}`이 "없음"이 아닐 때만 NotebookLM 호출.

**프롬프트 1 (NotebookLM 질의):**
```
NotebookLM의 {notebooklm_notebook} 노트북에 다음을 질의한다:
"{product_category} 시장의 주요 트렌드 3가지, 시장 규모, 성장 전망. 소스 근거를 명시해줘."
```

**프롬프트 2 (파일 저장):**
```
지금까지 분석한 불만 TOP 10, 만족 TOP 10, 교차 인사이트 3가지,
NotebookLM 시장 트렌드(있는 경우)를 종합해서
output/review_keywords_analysis.md로 정리해 저장한다.
```

**산출물:** `output/review_keywords_analysis.md`

**사용자 검증 ★:** 교차 인사이트가 우리 시장 맥락에 맞는지 확인 후 다음 단계 진행

# 3단계. 타겟 페르소나 도출

## Step 3-1. 리뷰 기반 페르소나 3개 ★

**프롬프트:**
```
output/review_keywords_analysis.md와 input/{product_slug}_reviews.md를 읽고,
리뷰에서 나타나는 구매자 패턴을 분석해서
서로 다른 유형의 타겟 페르소나 3개를 도출한다.

각 페르소나에 포함:
- 이름, 나이대, 직업, 라이프스타일
- 구매 동기, 핵심 니즈, 가장 큰 불만 포인트

각 항목이 어떤 리뷰 패턴에서 나왔는지 근거를 함께 표시한다.
```

**프롬프트의 결정적 두 요소 (절대 빠뜨리지 말 것):**
1. "리뷰에서 나타나는 구매자 패턴을 분석해서" — 교과서 페르소나 방지
2. "근거를 달아줘" — 허구 vs 실제 구분 가능

**사용자 검증 ★:** 페르소나 3개가 실존할 법한지 확인. 부적절하면 재실행.

---

## Step 3-2. 1번 페르소나 구매 여정 + 페르소나 파일 저장

**프롬프트 1:**
```
첫 번째 페르소나의 구매 여정을 단계별로 정리한다.
인지 → 탐색 → 비교 → 구매 → 사용 → 재구매.
각 단계의 채널, 찾는 정보, 감정, 리뷰 근거를 포함한다.
```

**프롬프트 2:**
```
3개 페르소나와 1번 페르소나의 구매 여정을 종합해서
output/target_personas.md로 저장한다.
각 페르소나별 프로필·구매 동기·핵심 니즈·불만 포인트·추천 마케팅 메시지 포함.
```

**산출물:** `output/target_personas.md`

# 4단계. 종합 리포트 + 리서치 브리프

## Step 4-1. 6섹션 종합 리포트 ★

**프롬프트:**
```
{notebooklm_notebook} 노트북에서 시장 규모·성장률·트렌드를 가져오고
(없으면 input/market_research_data.md만 사용),
output/review_keywords_analysis.md, output/target_personas.md,
input/market_research_data.md를 모두 참고해 종합 시장 분석 리포트를 작성한다.

구조:
① 시장 개요 (NotebookLM 또는 시장 자료 근거)
② 경쟁 환경 (스펙 비교 + 리뷰 강약점)
③ 고객 분석 (페르소나 + 키워드 TOP 5)
④ 기회 영역 3가지
⑤ 리스크 3가지
⑥ 전략 제안 5가지 (상세페이지·광고·제품 개선 포함)

모든 분석에 데이터 근거를 명시한다.
output/market_analysis_report.md로 저장한다.
```

**산출물:** `output/market_analysis_report.md`

**사용자에게 안내해야 할 경고:**
> 이 리포트는 시작점이지 결론이 아닙니다. 시장 규모 등 핵심 수치는 원본 소스와 대조하시고, 인사이트가 우리 브랜드 상황에 맞는지는 시장을 아는 분이 직접 검증해 주세요.

---

## Step 4-2. 리서치 브리프 압축

**프롬프트:**
```
output 폴더의 review_keywords_analysis.md, target_personas.md,
market_analysis_report.md를 읽고,
다른 에이전트가 바로 참고할 수 있는 리서치 브리프를 작성한다.

포함 항목:
① 타겟 고객 한줄 요약
② 핵심 구매 동기 3가지
③ 가장 중요한 차별화 포인트 3가지
④ 반드시 강조할 키워드 5개
⑤ 절대 쓰지 말아야 할 표현 3가지 (왜 피해야 하는지 이유 포함)
⑥ 경쟁사 대비 우위 요소 3가지
⑦ 시장 규모 및 성장률 핵심 수치

분량은 1~2페이지로 압축. output/research_brief.md로 저장한다.
```

**산출물:** `output/research_brief.md`

**사용자 검증 ★:** ⑤번 "쓰지 말아야 할 표현"이 적절한지 검토 (예: "가성비 최고" 같은 범용 표현, "전문가용" 같은 타겟 불일치 등)

## Step 4-3. (선택) 활용 맛보기

**조건:** 사용자가 "상세페이지 만들어줘" 또는 "제안서 만들어줘"라고 추가 요청한 경우에만 실행. 자동 진행 금지.

```
output/research_brief.md를 참고해서 {product_name} 상세페이지 기획안 초안을 만든다.
```

# 스킬 종료 시 사용자에게 보여줄 요약

```
✅ 시장조사 4단계 파이프라인 완료

생성된 파일:
- input/{product_slug}_reviews.md       — 리뷰 원본 데이터
- input/market_research_data.md         — 시장 자료 (해당 시)
- output/review_keywords_analysis.md    — 키워드 분석 + 교차 인사이트
- output/target_personas.md             — 타겟 페르소나 3개 + 구매 여정
```

# 에러 / 예외 처리 규칙

| 상황 | 처리 |
|---|---|
| `input/`, `output/` 폴더가 없음 | STEP 0-A에서 위치 결정 후 생성 |
| `input/output` 폴더 쌍이 여러 곳에 있음 | STEP 0-A에서 후보 보여주고 사용자 선택 |
| 리뷰 이미지가 첨부되지 않음 | 사용자에게 첨부 요청, 받을 때까지 대기 |
| 추출된 리뷰가 50개 미만 | 사용자에게 알리고 추가 진행/중단 결정 |
| NotebookLM 노트북이 "없음" | 시장 트렌드 단계 스킵, 로컬 파일 기반으로만 진행 |
| 사용자가 "다시" 요청 | 직전 Step만 재실행, 이전 산출물은 유지 |
| 사용자 검증 ★에서 "수정" 요청 | 구체적 수정 사항 묻고 해당 부분만 보강 |
