# /start — NotebookLM 에이전트 셋업 & 시연 마법사

`/start`가 호출되면 사용자가 처음 진입한 것으로 보고 **1단계**부터 시작합니다.
이미 진행 중이라면 사용자의 마지막 입력(`다음`, `처음부터`, `끝`)에 따라 분기합니다.

## 진행 규칙 (반드시 준수)

- **한 번의 응답에 한 단계만** 출력. 두 단계를 합쳐서 보내지 않음
- 명령은 Bash 도구로 실행 → 성공 조건 검증 → 실패 시 1회 재시도 → 여전히 실패면 에러를 사용자에게 그대로 보고
- 응답 마지막 줄: ``➡️  다음 단계는 `다음` 또는 `next` 입력``
- `다음`/`next` → 다음 단계, `처음부터`/`restart` → 1단계, `끝`/`done` → 종료 메시지 후 마법사 종료
- **비유 금지**. "마치 ~처럼", "식당/문/주방" 같은 표현 사용하지 말 것
- 모든 명령은 저장소 루트(`notebooklm-agent-github/`)에서 실행
- 5~7단계는 **사용자가 NotebookLM 웹에서 직접 수행**. 마법사는 안내만 제공하고 명령은 실행하지 않음

---

## A. MCP 설치 (1~4) — 약 3분

### 1단계: 환경 점검
표시: "1/13 환경 점검 — Node 18+ / npx 확인"
실행: `node scripts/check-env.mjs`
성공: 종료 코드 0 + stdout에 "환경 점검 통과"
실패: 출력된 안내(Node 업그레이드 / npm 재설치)를 그대로 사용자에게 전달

### 2단계: NotebookLM MCP 캐시
표시: "2/13 npx 캐시 — 첫 실행은 1~2분 소요"
실행: `npx -y notebooklm-mcp@latest --help` (Bash timeout 180000)
성공: stdout에 도구 도움말 출력
의미: npm 캐시에 패키지 다운로드. 이후 모든 명령은 캐시에서 즉시 실행됨.

### 3단계: Google 계정 인증
사용자 안내: "자동화 전용 Google 계정 사용을 권장합니다(개인 계정은 보안 위험). 준비됐으면 `다음` 입력."
실행: `node scripts/setup-auth.mjs` (Bash timeout 360000)
브라우저가 열리면 사용자가 직접 로그인 → 쿠키가 Chrome 프로필에 저장됨
성공: stdout에 `✅ 인증 완료` 또는 `authenticated: true`

### 4단계: 헬스체크 (= /mcp 역할)
표시: "4/13 연결 상태 확인"
실행: `node scripts/health-check.mjs`
성공: status `ok` + 인증 상태 정상 (노트북 목록은 비어있어도 OK)

## B. 노트북 준비 (5~7) — 약 4분 (수동)

### 5단계: NotebookLM 웹에서 노트북 생성
이 단계부터 7단계까지는 **사용자가 NotebookLM 웹**(notebooklm.google.com)에서 직접 수행. 마법사는 명령을 실행하지 않고 안내만 제공.

사용자에게 다음을 그대로 출력:
```
NotebookLM 웹(notebooklm.google.com)에 접속해서:

1. 우측 상단 "새 노트북" 클릭
2. 노트북 이름 입력 (예: "크래프트볼트 시장분석")
3. 빈 노트북이 생성됨

다음 단계에서 PDF를 첫 소스로 추가합니다.
```

생성되면 `다음` 입력 안내.

### 6단계: PDF 소스 추가 ⭐ 시연 핵심
PDF는 출처 인용이 가장 명확하게 드러나는 소스 타입. 시연 효과가 가장 큼.
사용자에게 다음을 그대로 출력:

```
PDF 1~3개를 노트북에 추가합니다.

🎯 즉시 사용 가능한 시연용 PDF (저장소에 포함됨):
  📄 input/★ 최종 전기톱 사용설명서 시안.pdf
     크래프트 볼트 8인치 무선 전기톱 공식 사용설명서 (18p, 약 1.6MB)
     → 제품 사양 / 안전 / 유지관리 / FAQ / 보증 정보 포함
     → 이 PDF만으로도 시연 진행 가능

📄 추가 PDF 후보 (선택):
  - 산업 보고서 / 시장 조사 자료 (예: 한국기계산업진흥회 무선전동공구 보고서)
  - 경쟁사 카탈로그 / 백서
  - 학술 논문 / 협회 통계 자료
  - 네이버 블로그 등 iframe 차단 페이지 → 브라우저에서 Cmd+P / Ctrl+P → "PDF로 저장"
    (블로그를 PDF로 변환하면 7단계 URL 우회로 활용 가능)

📤 추가 방법:
  1. 노트북 좌측 "+ 소스 추가" 클릭
  2. "파일 업로드" → input/ 폴더의 PDF 선택 (또는 드래그)
  3. 처리 완료까지 30초~1분 대기 (인덱싱 시간)
  4. 좌측 소스 패널에 파일명이 표시되면 성공

⚠️ 제한: 파일당 200MB, 노트북당 최대 50개 소스
✅ 권장: 저장소 PDF + 추가 1~2개 = 총 2~3개로 출처 다양성 확보
```

PDF가 1개 이상 추가되었으면 `다음` 입력 안내.
사용자가 PDF 없이 진행하려 하면: "PDF 없이도 동작하지만, 8단계 응답에서 소스 인용 효과가 약해집니다. 그래도 진행할까요?" 확인.

### 7단계: 추가 소스 반자동 큐레이션 — YouTube + 산업 자료
**역할 분담**: 에이전트가 검색·큐레이션을 자동 수행하고, NotebookLM 등록은 사용자가 수동으로 진행.

#### 7-1. 에이전트 자동 작업 — WebSearch 6회 실행
다음 6개 쿼리를 **WebSearch 도구로 병렬 실행**:

YouTube 트랙 (3쿼리):
- `크래프트볼트 전기톱 리뷰 youtube`
- `무선 미니 전기톱 비교 youtube`
- `브러시리스 전기톱 사용기 youtube`

산업 자료 트랙 (3쿼리):
- `한국 무선 전동공구 시장 보고서 PDF`
- `전기톱 시장 동향 산업통상자원부`
- `cordless chainsaw market report 2024`

#### 7-2. 큐레이션 결과 출력
검색 결과에서 **각 트랙당 최대 5건씩 추려서** 다음 형식으로 사용자에게 표시:

```
## 🎥 YouTube (최대 5건)

1. **[제목 또는 분류 태그] 영상 제목**
   https://youtube.com/watch?v=...
   요약: 한 줄 핵심 내용
   추천도: ★~★★★★★ (자막 유무 / 시연 적합성 기준)

(2~5번 동일 형식)

## 🏭 산업 자료 (최대 5건)

1. **[발행처] 자료 제목**
   https://...
   요약: 한 줄 핵심 내용
   추천도: ★~★★★★★ (PDF 직접 다운로드 가능 / 데이터 신선도 기준)

(2~5번 동일 형식)
```

#### 7-3. 사용자 수동 작업 안내
큐레이션 결과 아래에 다음 메시지 그대로 출력:

```
위 후보 중 마음에 드는 것을 NotebookLM에 직접 추가하세요.

📍 추가 위치: 노트북 좌측 "+ 소스 추가" 버튼
🔗 YouTube URL → "YouTube" 탭에 붙여넣기
🔗 일반 URL → "웹사이트" 탭에 붙여넣기
⚠️ 네이버 블로그 / iframe 차단 페이지: 6단계로 돌아가서 PDF 인쇄 후 업로드 우회
✅ 권장: YouTube 1~2개 + 산업 자료 1~2개 (총 2~4개)

📐 소스 운영 원칙
  - 양보다 질: 핵심 자료 5~10개로 충분
  - 주제 통일: 다른 주제 자료는 별도 노트북으로
  - 출처 확인: 소스 품질 = 답변 품질

작업이 끝나면 노트북 URL을 복사하세요.
URL 형식: https://notebooklm.google.com/notebook/...

`다음` 입력 시 URL과 노트북 이름을 함께 알려주세요.
```

#### 7-4. 추천도 평가 기준 (에이전트 내부 규칙)
- **YouTube ★★★★★**: 자막 있음 + 제품/시장 직접 다룸 + 30분 이내
- **YouTube ★★★**: 자막 있음 + 간접 관련 (예: 전동공구 일반론)
- **YouTube ★★**: 자막 없음 또는 광고성
- **산업 자료 ★★★★★**: PDF 직접 다운로드 가능 + 2024년 이후 + 한국 시장
- **산업 자료 ★★★**: 웹페이지 형태 + 2022년 이후
- **산업 자료 ★★**: 오래된 자료 (2021년 이전) 또는 일반 블로그

#### 7-5. 진입 조건
사용자가 노트북 URL을 알려주면 → 8단계 진입.

## C. 에이전트에서 질문 (8~11) — 약 5분

### 8단계: 노트북 등록
사용자에게서 받은 정보로 실행:
`node scripts/setup-notebook.mjs "<URL>" "<이름>" "<설명>" "<태그1,태그2>"`

예:
```
node scripts/setup-notebook.mjs \
  "https://notebooklm.google.com/notebook/abc..." \
  "크래프트볼트 시장분석" \
  "무선 전동공구 시장 트렌드와 경쟁사 분석" \
  "시장,경쟁,트렌드"
```
성공: stdout에 `success: true` 또는 노트북 ID

### 9단계: 기본 노트북 지정
실행: `node scripts/select-notebook.mjs "<8단계에서 등록한 이름 또는 URL>"`
성공: `🎯 선택: <이름> (<id>)` 출력
의미: 이후 ask 호출 시 노트북 URL 생략 가능.

### 10단계: 첫 질문 (소스 귀속 포함) ⭐ 시연 클라이맥스
실행:
```
node scripts/ask.mjs \
  "무선 전동공구 시장의 주요 트렌드 3가지를 정리해줘. 각 트렌드마다 근거가 되는 소스 파일명을 명시해줘." \
  output/q1_trends.md
```
성공: `output/q1_trends.md` 생성. Read로 1~30줄 미리보기 표시.
주안점: NotebookLM 응답에 `[파일명.pdf]` `[블로그.com]` 같이 다양한 출처가 인용되는 것을 사용자에게 명시적으로 보여줌. 6~7단계에서 PDF/URL/텍스트를 모두 등록했다면 인용 다양성이 가장 잘 드러나는 시점.

### 11단계: 후속 질문 (멀티패스 1→2→3)
사용자에게 안내: "같은 노트북에 대해 3회 연속 질문하여 단계적으로 깊이를 확보합니다."

세 번 연속 실행:

**1차 — 개요**
```
node scripts/ask.mjs \
  "지금까지 확인한 트렌드 중 가장 임팩트 큰 것 1가지를 골라 시장 데이터로 뒷받침해줘." \
  output/q2_overview.md
```

**2차 — 심층 (차별화 포인트)**
```
node scripts/ask.mjs \
  "경쟁사 대비 브러시리스 모터 기술에서 우리가 차별화할 수 있는 포인트 3개를 도출해. 각 포인트마다 소스 근거 포함." \
  output/q3_diff.md
```

**3차 — 액션 아이템**
```
node scripts/ask.mjs \
  "위 분석을 종합해서 다음 분기에 실행할 액션 아이템 3개를 우선순위 순으로 제안해." \
  output/q4_actions.md
```

성공: 3개 파일 모두 생성. 각 파일 1~10줄 미리보기 표시.

## D. 자동 리포트 (12) — 약 3분

### 12단계: 통합 리포트 자동 생성
사용자에게 주제 확인:
> 자동 리포트의 주제를 확인하세요(기본: "무선 전동공구 시장"). 변경하려면 입력하세요.

실행:
```
node scripts/auto-report.mjs "<주제>" output/market_report.md
```
(스크립트가 6개 표준 질문을 1회 spawn에서 순차 실행 → 섹션별 통합 .md 저장)

성공: `output/market_report.md` 생성. Read로 1~50줄 미리보기 표시.
구조: 시장 개요 / 주요 트렌드 3가지 / 경쟁 환경 / 기회 / 리스크 / 액션 아이템 6섹션.

## E. 마무리 (13)

### 13단계: 산출물 정리
다음 형식으로 출력:
```
✅ 셋업 + 시연 완료

생성된 파일:
- output/q1_trends.md      (첫 질문 — 트렌드 3가지, 다중 소스 인용)
- output/q2_overview.md    (멀티패스 1차 — 임팩트 트렌드)
- output/q3_diff.md        (멀티패스 2차 — 차별화 포인트)
- output/q4_actions.md     (멀티패스 3차 — 액션 아이템)
- output/market_report.md  (자동 리포트 — 6섹션 통합)

이 데이터는 다음에 재활용할 수 있습니다:
- 상세페이지 기획안 초안
- 제안서 시장 현황 섹션
- 콘텐츠 / 마케팅 카피

다시 처음부터 진행하려면 /start
```

마법사 종료.
